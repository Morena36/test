── QUÉ HACE LA PÁGINA ──────────────────────────────────────────
Una página oscura y elegante que:
1. Fade-in desde negro al cargar (suave, 2 segundos)
2. Muestra un video (video.mp4) en formato 16:9, responsive
3. Botón de sonido 🔇/🔊 (el video arranca muted por política del navegador)
4. Flecha animada apuntando hacia abajo
5. Botón grande "Hacé click acá para ver la sorpresa"
6. Al hacer click se despliegan 2 opciones (tarjetas):
   - 🎬 Ir al cine → abre WhatsApp con mensaje pre-escrito
   - 🍕 Salir a comer → abre WhatsApp con mensaje pre-escrito
7. 2 opciones extra ocultas (class="card-hidden") listas para activar


── MENSAJES DE WHATSAPP ────────────────────────────────────────
Cine:  "¿Vamos al cine? 🍿"
Comer: "¿Comemos algo juntos? 🍕"
(URL encode directo en el onclick de cada botón)


── ARCHIVOS DEL PROYECTO ───────────────────────────────────────

video.mp4      → debe estar en la MISMA carpeta que el HTML


── DISEÑO Y ESTÉTICA ───────────────────────────────────────────
- Tema oscuro (#0a0a0a fondo)
- Tipografías: Playfair Display (serif elegante) + DM Sans (sans-serif limpia)
- Colores: rosa #f0436a, dorado #e8c97a, blanco roto #f5f0eb
- Fondo con gradientes radiales sutiles (rojo oscuro arriba, azul oscuro abajo)
- Animaciones CSS puras: fadeDown, fadeUp, bounce, arrowBounce
- Tarjetas con colores diferenciados: dorado para cine, naranja para comer
- Entrada suave: overlay negro que hace fade-out al cargar


── VARIABLES CSS PARA AJUSTAR EL VIDEO ─────────────────────────
--video-max-width: 420px;   → ancho máximo en móvil
--video-ratio: 16/9;        → proporción (cambiar a 9/16 si el video es vertical)
--video-radius: 20px;       → bordes redondeados


── DESKTOP (media query 768px+) ────────────────────────────────
El video se agranda automáticamente a 700px de ancho.
El container y controles también se expanden.


── SONIDO ──────────────────────────────────────────────────────
El video arranca MUTED (obligatorio para autoplay en Chrome/Safari móvil).
El botón 🔇 Sonido llama a toggleSonido() que hace vid.muted = !vid.muted.
No hace falta archivo de audio separado: el sonido está dentro del mp4.

── TÉCNICA DE EDICIÓN RÁPIDA (str_replace) ─────────────────────
En lugar de reescribir el archivo entero, se edita solo la línea exacta:

1. Buscar el texto EXACTO que está en el archivo (con grep -n si hace falta)
2. Usar str_replace con old_str = texto original, new_str = texto nuevo
3. Si falla "string not found": usar view para ver el archivo actualizado
   (cada edición invalida el contexto anterior)
4. Para borrar: new_str vacío o solo lo que queda
5. Para archivos en /mnt/user-data/uploads (read-only): copiar primero a /home/claude/

Ejemplo:
  old_str: <video src="video.mp4" autoplay loop muted playsinline>
  new_str: <video src="video.mp4" autoplay loop muted playsinline id="vid">

Esto es 10x más rápido que regenerar el HTML completo.
